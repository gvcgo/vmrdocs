## 需要使用source刷新环境变量?

VMR为powershell贴心地定制了**"source"**命令，即使在Windows下，用户也能愉快的使用source来刷新环境变量.
可以在下面的路径下查看具体定制的内容，注意不要随意改动.
```text
$HOME\Documents\WindowsPowerShell\profile.ps1
```

## 在Windows下有一些应用无法使用？

例如agg, php, 等等. 可能是因为你的系统缺少Microsoft Visual C++ Redistributable. 你可以从[这里](https://learn.microsoft.com/en-us/cpp/windows/latest-supported-vc-redist?view=msvc-170)下载后安装.

## VMR安全可靠吗?

必须的. 所有的SDK都来自**官网**, 或者 **github的releases**, 或者 **Anaconda Forge**. 
另外，VMR不提供源码编译功能，因为这样做不利于稳定性和跨平台. 
如果你有任何疑问，可以去VMR的[版本仓库](https://github.com/gvcgo/vsources)查看具体下载链接.

## 关于数据库安装

VMR只负责安装，不进行配置。你可以到SDK安装目录(默认是$HOME/.vmr/versions/)下面查看，然后自行配置用户名、密码、数据库文件位置、开机启动等等。
